Metric aggregations is divided into 2 subgroups:

* single-value numeric metric aggregations. Outputs a single value, which can be the sum of numbers or an average value.
* multi-value numeric metric aggregations. Yields multiple values.

If we want to aggregate all docs, we don't need to explicitly add a query (`match_all`). 

## Calculating statistics with sum, avg, min, and max aggregations

```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "total_sales": {
      "sum": {
        "field": "total_amount"
      }
    }
  }
}
```

Response

```
{
  "took" : 10,
  "timed_out" : false,
  "_shards" : {
    "total" : 1,
    "successful" : 1,
    "skipped" : 0,
    "failed" : 0
  },
  "hits" : {
    "total" : {
      "value" : 1000,
      "relation" : "eq"
    },
    "max_score" : null,
    "hits" : [ ]
  },
  "aggregations" : {
    "total_sales" : {
      "value" : 109209.61
    }
  }
}
```

## Retrieving the number of distinct values

```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "total_salesmen": {
      "cardinality": {
        "field": "salesman.id"
      }
    }
  }
}
```

```
{
  "took" : 49,
```

## Using `stats` aggregation for common statistics

```
GET /order/_search
{
  "size": 0,
  "aggs": {
    "amount_stats": {
      "stats": {
        "field": "total_amount"
      }
    }
  }
}
```

```
{
  "took" : 9,
  "timed_out" : false,
  "_shards" : {
    "total" : 1,
    "successful" : 1,
    "skipped" : 0,
    "failed" : 0
  },
  "hits" : {
    "total" : {
      "value" : 1000,
      "relation" : "eq"
    },
    "max_score" : null,
    "hits" : [ ]
```
